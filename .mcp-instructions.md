# Instruções para IA - C# MCP Tool

## Visão Geral
Este é um servidor MCP (Model Context Protocol) que fornece 40 tools para automação de desenvolvimento .NET. O servidor atua como proxy para uma API .NET onde toda a execução acontece através de um **Pipeline de Behaviors**.

## Arquitetura
```
IA (Você) → MCP Server (Node.js Proxy) → API .NET (Pipeline) → Execução Real
```

## Pipeline de Execução

Todas as operações passam por um pipeline de behaviors na seguinte ordem:

1. **TransactionBehavior** - Cria snapshot antes da operação e executa rollback em caso de falha
2. **ValidationBehavior** - Valida parâmetros de entrada e caminhos
3. **SandboxBehavior** - Restringe acesso a arquivos fora do workspace
4. **SecurityBehavior** - Valida comandos permitidos e detecta padrões suspeitos
5. **LoggingBehavior** - Registra logs estruturados de execução
6. **ErrorHandlingBehavior** - Trata erros e exceções
7. **ExecutionBehavior** - Executa a ação real
8. **DiffBehavior** - Gera diff antes/depois para operações de escrita

### Características do Pipeline
- **Transacional**: Rollback automático em caso de falha
- **Seguro**: Sandbox de arquivos e whitelist de comandos
- **Observável**: Logs estruturados e diffs de alterações
- **Extensível**: Behaviors podem ser adicionados ou removidos
- **Dry-Run**: Suporte a simulação sem execução real

## Inicialização Automática

**IMPORTANTE: Ao iniciar uma nova sessão, execute o seguinte fluxo:**

1. **Verificar estado de instruções:**
   ```json
   {
     "action": "read_file",
     "payload": {
       "path": ".mcp-state.json",
       "workspace": "CAMINHO_DO_MCP_SERVER"
     }
   }
   ```

2. **Se `instructions_read` for `false`:**
   - Ler este arquivo (`.mcp-instructions.md`)
   - Atualizar estado com:
     ```json
     {
       "action": "update_file",
       "payload": {
         "path": ".mcp-state.json",
         "content": "{\"instructions_read\": true, \"last_read\": \"DATA_ATUAL\", \"version\": \"1.0.0\"}",
         "workspace": "CAMINHO_DO_MCP_SERVER"
       }
     }
   ```
   - Continuar com o trabalho

3. **Se `instructions_read` for `true`:**
   - Pular leitura de instruções
   - Continuar com o trabalho

## Pré-requisitos Importantes
1. **API .NET deve estar rodando** em `http://localhost:5000` antes de usar qualquer tool
2. **Sempre use o parâmetro `workspace`** quando trabalhando em projetos específicos
3. **Não gere código no chat** - use as tools MCP para criar/modificar arquivos
4. **Execute verificação de estado** ao iniciar cada sessão
5. **Todas as operações são transacionais** - rollback automático em caso de falha
6. **Acesso a arquivos é sandboxed** - apenas arquivos dentro do workspace são acessíveis

## Funcionalidades Avançadas

### Roslyn Engine
O sistema inclui um motor de análise e refatoração de código C# usando Roslyn:

- **RefactorService**: Encontrar, substituir, alterar métodos, remover membros, inserir código
- **SemanticAnalysisService**: Análise semântica com Compilation + SemanticModel
- **CodeFormatterService**: Formatação automática de código

### Capabilities
Sistema de capabilities para operações especializadas:

- **CodeAnalysisCapability**: Análise de código C# (erros, warnings, métricas)
- **RefactorCapability**: Refatoração via Roslyn (remover usings, formatar, remover métodos)
- **GenerationCapability**: Geração de código (classes, interfaces, records, enums)
- **CliExecutionCapability**: Execução segura de comandos CLI com whitelist

### Planner + Validator Loop
Sistema de planejamento e execução de múltiplas operações:

- **PlanStep**: Passos de execução com dependências
- **PlanExecutor**: Executor com validação e rollback automático
- Suporte a dependências entre steps
- Continuação em caso de erro (configurável)

### Controle de Concorrência
- **FileLockService**: Lock por arquivo para operações simultâneas
- Previne race conditions em operações de arquivo

### Idempotência
- **IdempotencyService**: Evita operações duplicadas
- Cache de resultados por 5 minutos
- Detecção automática de operações em progresso

## Gerenciamento de Contexto

### Quando Usar Estratégia de Contexto
Use a estratégia de gerenciamento de contexto quando:
- Contexto atingir > 50% do limite disponível
- Múltiplas tarefas acumuladas
- Respostas ficarem mais lentas
- IA mencionar limite de contexto

### Estratégia Híbrida

**1. Memory System (Curto Prazo)**
- Use `append_memory` para salvar resumos rápidos
- Use `read_memory` para recuperar contexto compactado
- Formato: veja `templates/summary-template.md`

**2. Arquivamento por Sessão (Médio Prazo)**
- Estrutura: `context/sessions/YYYY-MM-DD/session-XXX.md`
- Use `create_file` para arquivar contexto completo
- Use `search_in_files` para buscar contexto específico

**3. Pipeline Automático**
- Use `execute_pipeline` com `pipelines/context-compact.json`
- Orquestra compactação e arquivamento automaticamente

### Exemplo de Compactação de Contexto
```json
{
  "action": "execute_pipeline",
  "payload": {
    "pipeline": {
      "name": "context-compact",
      "description": "Compacta contexto atual",
      "steps": [
        {
          "id": "save-summary",
          "action": "append_memory",
          "parameters": {
            "content": "2026-04-25: Implementada feature X. Arquivos: A.cs, B.cs.",
            "workspace": "CAMINHO_DO_PROJETO"
          }
        },
        {
          "id": "create-session-dir",
          "action": "create_directory",
          "parameters": {
            "path": "context/sessions/2026-04-25",
            "workspace": "CAMINHO_DO_PROJETO"
          }
        },
        {
          "id": "archive-context",
          "action": "create_file",
          "parameters": {
            "path": "context/sessions/2026-04-25/session-001.md",
            "content": "# Contexto Completo\n\n[Conteúdo do contexto...]",
            "workspace": "CAMINHO_DO_PROJETO"
          }
        }
      ]
    }
  }
}
```

## Parâmetro Workspace

**Sempre use o parâmetro `workspace`** para especificar o diretório do projeto:
```json
{
  "action": "create_file",
  "payload": {
    "path": "src/Models/User.cs",
    "content": "namespace App.Models;\n\npublic class User\n{}",
    "workspace": "C:/MeuProjeto"
  }
}
```

## Fluxo de Trabalho Recomendado

1. **Ler memory.md** para entender contexto atual
2. **Usar tools MCP** para todas as ações (não gere código no chat)
3. **Registrar em memory.md** após cada tarefa importante
4. **Compactar contexto** quando atingir limite
5. **Usar workspace** para especificar diretório do projeto

## Troubleshooting

**API não responde:**
- Verifique se API .NET está rodando em `http://localhost:5000`
- MCP Server é proxy e não funciona sem API ativa

**Comandos falham:**
- Verifique se está usando parâmetro `workspace` correto
- Execução acontece na API .NET, não no Node.js

**Contexto grande:**
- Use estratégia de gerenciamento de contexto
- Compacte e arquive contexto periodicamente
